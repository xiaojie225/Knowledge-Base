## 单位分类

CSS 单位分为两大类：

- **绝对单位**：px（像素）- 长度固定不变
- **相对单位**：百分比、em、rem、vw、vh - 长度相对于其他参照物

这是 CSS 基本功的重要考察点，这些都是必备且常用的单位。

## px（像素）

- **定义**：基本单位，绝对单位
- **特点**：单位长度固定，不会随其他因素改变
- **用途**：适合需要精确控制尺寸的场景

## 百分比（%）

- **定义**：相对单位
- **参照**：相对于**父元素**的宽度比例
- **经典应用场景**：元素在父元素中的垂直和水平居中

### 居中布局示例

让一个元素在其父元素中绝对定位后水平垂直居中：

```css
.container {
  width: 设定宽度;
  height: 设定高度;
  position: relative;
  background: 背景色;
}

.box {
  width: 设定宽度;
  height: 设定高度;
  position: absolute;
  top: 50%;
  left: 50%;
  margin-top: -50px;  /* 负的元素高度的一半 */
  margin-left: -50px; /* 负的元素宽度的一半 */
}
```

**原理说明**：
- `top: 50%` 和 `left: 50%` 使元素的**左上角**定位在父元素的中心点
- 相对于父元素高度的 50% 和宽度的 50%
- 使用负的 margin 将元素向回拉，使其真正居中
- `margin-top` 和 `margin-left` 分别设置为元素自身高度和宽度的负一半

## em 单位

- **定义**：相对单位
- **参照**：相对于**当前元素**的 font-size
- **注意**：不是相对于父元素，而是当前元素本身

### 实验验证

```css
.container {
  font-size: 20px;
}

.p1 {
  font-size: 40px;
  text-indent: 2em; /* 实际为 80px */
}

.p2 {
  font-size: 20px; /* 继承父元素 */
  text-indent: 2em; /* 实际为 40px */
}
```

**结论**：
- `p1` 的 `text-indent` 是 80px（40px × 2）
- `p2` 的 `text-indent` 是 40px（20px × 2）
- 证明 em 相对于当前元素的 font-size，而非父元素

## rem 单位

- **定义**：root em，相对单位
- **参照**：相对于**根节点**（html 元素）的 font-size
- **优势**：便于实现响应式设计

### 响应式布局应用

通过媒体查询为不同屏幕宽度设置不同的根字体大小：

```css
/* 小屏幕手机 */
@media (max-width: 374px) {
  html {
    font-size: 较小值;
  }
}

/* 中等屏幕 */
@media (min-width: 375px) and (max-width: 413px) {
  html {
    font-size: 16px; /* 正常值 */
  }
}

/* 大屏幕 */
@media (min-width: 414px) {
  html {
    font-size: 17.6px; /* 较大值 */
}
```

**应用场景**：
- 移动端适配
- 根据屏幕宽度自动调整所有使用 rem 的元素
- 实现响应式的字体和尺寸缩放
- 例如：iPhone 8 Plus（414px 宽）会使用 17.6px 的根字体

## vw 和 vh 单位

- **vw（viewport width）**：屏幕宽度的 1%
- **vh（viewport height）**：屏幕高度的 1%
- **含义**：w 代表 width，h 代表 height

### 示例代码

```css
.div1 {
  width: 10vw;  /* 屏幕宽度的 10% */
  height: 10vh; /* 屏幕高度的 10% */
}
```

**实际计算**：
- 假设屏幕宽度 350px，高度 427px
- `10vw` = 35px
- `10vh` = 42.7px

## vmin 和 vmax 单位

- **vmin**：取 vw 和 vh 中的**最小值**
- **vmax**：取 vw 和 vh 中的**最大值**

### 示例说明

以手机竖屏为例（高度 > 宽度）：
- 宽度：350px（35vw）
- 高度：427px（42.7vh）

```css
.div2 {
  width: 10vmax;  /* 取最大值：42.7px */
  height: 10vmax; /* 取最大值：42.7px */
}

.div3 {
  width: 10vmin;  /* 取最小值：35px */
  height: 10vmin; /* 取最小值：35px */
}
```

**验证结果**：
- `vmax` 取的是高度值 42.7px（更大）
- `vmin` 取的是宽度值 35px（更小）

## 应用场景总结

理解不同单位存在的**根本原因**和**应用场景**至关重要：

1\. **px**：精确控制，固定尺寸
2\. **百分比**：相对父元素布局，流式布局
3\. **em**：相对当前元素字体大小，局部缩放
4\. **rem**：响应式设计，统一缩放
5\. **vw/vh**：视口相关布局，全屏元素
6\. **vmin/vmax**：适应不同屏幕方向

如果不理解这些单位的使用场景和设计初衷，就无法真正掌握它们。要思考：为什么 CSS 需要这么多不同的单位?它们各自解决什么问题?
